## AI 문항 생성

### [ISSUE-001] 나눗셈 상 난이도 — 나누어떨어지지 않는 stem 생성 ✅ 해결됨
- **현상**: "28 ÷ 4 = 7모둠, 절반으로 줄이면 → 한 모둠 8명" 같은 문항에서
  실제로 28 ÷ 8 = 3.5로 나누어 떨어지지 않아 stem 자체가 성립 불가.
- **원인**: Pass 2(검산)가 "나머지 ≠ 0이면 문항 제거" 조건을 놓침.
- **해결** (`claude/question-accuracy-pass2`):
  1. SYSTEM 프롬프트 [계산 정확성] 섹션에 나눗셈 제약 명시 (방안 1)
  2. VERIFY_SYSTEM_PROMPT에 "나누어 떨어지지 않는데 몫만 묻는 경우 → 제거" 명시 (방안 2)
  3. `getUnitConstraints("나눗셈")`: USER 프롬프트에 단원별 주의사항 동적 주입 (방안 3 경량화)
  4. Pass 2 제거분 재생성 루프 (최대 2라운드) 추가
