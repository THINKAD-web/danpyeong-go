## AI 문항 생성

### [ISSUE-001] 나눗셈 상 난이도 — 나누어떨어지지 않는 stem 생성
- **현상**: "28 ÷ 4 = 7모둠, 절반으로 줄이면 → 한 모둠 8명" 같은 문항에서
  실제로 28 ÷ 8 = 3.5로 나누어 떨어지지 않아 stem 자체가 성립 불가.
- **원인**: Pass 2(검산)가 "나머지 ≠ 0이면 문항 제거" 조건을 놓침.
- **개선 방안** (택일):
  1. SYSTEM 프롬프트에 "나눗셈 문항의 수는 나누어 떨어지는 수만 사용할 것" 제약 추가
  2. Pass 2 VERIFY 프롬프트에 "나눗셈 연산이 포함된 stem은 나머지가 0인지 확인하고,
     나머지 ≠ 0이면 해당 문항 제거" 명시
  3. 서버 코드에서 생성 후 stem 텍스트 파싱 + 계산 검증 (복잡도 높음)
- **우선순위**: 낮음 (교사 검수 단계에서 걸러짐)
