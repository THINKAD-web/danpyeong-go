# 단평GO — Claude Code 작업 지시서

> 초등 단원평가 AI SaaS · 4~5주 MVP · Vercel 배포
> 이 문서를 Claude Code 세션의 기준 사양으로 사용한다.

---

## 0. 프로젝트 가드레일 (항상 준수)

- **스택 고정:** Next.js 15 (App Router, TS) / Tailwind + shadcn/ui / Prisma + Neon / Clerk / React Query + Zod / Framer Motion / Recharts. 임의 변경 금지.
- **배포 보호:** main 직접 push 금지. PR → Vercel Preview 확인 → 승인 후 머지. 다음 단계로 자동 진행 금지, 매 단계 승인 게이트.
- **학생은 로그인 없음:** Clerk 인증은 교사(TEACHER/ADMIN)만. 학생은 클래스 코드 + 이름으로 익명 응시(`Attempt`).
- **Question은 영구 저장:** AI 생성 문항도 DB에 저장(문제은행). `Test`는 `Question`을 참조만 하고 복제하지 않는다.
- **환경변수:** `DATABASE_URL`, `DIRECT_URL`(Neon), `CLERK_*`, AI 키는 `.env.local`. 절대 커밋 금지.

## TASK 0 — AI 모델 비교 검증 (Week 1, 코딩 전 필수)

서비스 가치 전체가 "AI가 3학년 수학 성취기준에 맞는 문항을 뽑느냐"에 달려 있다. 본 개발 전에 반드시 확인한다.

- 같은 단원(예: 3-1 "곱셈") + 같은 프롬프트로 **Grok vs Claude** 각각 객관식 5문항 생성
- 평가 기준: ① 교육과정 성취기준 적합성 ② 정답·해설 정확성 ③ 초등 3학년 난이도/표현 적절성 ④ 한국어 자연스러움
- 결과로 **기본 모델 확정**. 프롬프트만으로 부족하면 성취기준 텍스트를 컨텍스트로 주입하는 방식 채택.
- 산출물: `docs/AI_MODEL_DECISION.md` (비교 표 + 최종 결정 + 확정 프롬프트 템플릿)

## Phase 1 (Week 1~2) — 교사 코어 ★최우선

### 1-1. 프로젝트 셋업
- Next.js 15 + TS + Tailwind + shadcn/ui 초기화
- Prisma 연동(첨부 `schema.prisma` 사용) → `prisma migrate dev`
- Clerk 연동 (TEACHER 역할 미들웨어 보호)
- 시드: 3학년 수학 1~2학기 Subject/Unit + 성취기준(`achievementStandard`) 입력

### 1-2. AI 실시간 문제 생성
- 입력 폼: 단원 · 문제 수 · 난이도(하/중/상) · 유형(객관식/단답)
- 서버 액션에서 AI 호출 → 4지선다 + 정답 + 해설 생성
- **생성 결과는 즉시 DB 저장**(`source: AI`, `isReviewed: false`)
- 생성 후 **편집 화면 필수**: 문항 수정/삭제/추가, 정답 변경, 해설 편집
- 검수 완료 시 `isReviewed: true` 토글 → UX에 "교사 최종 검수" 명시

### 1-3. 테스트 생성/편집/배포
- 문제은행/방금 생성한 문항에서 골라 `Test` 조립(`TestQuestion` 순서·배점)
- 제목, 시간 제한, 순서 랜덤 설정
- 배포: `status: PUBLISHED` + `shareToken` 링크 / 클래스 코드 공유
- 마감: `status: CLOSED`

### 1-4. 클래스 관리 (기초)
- `Classroom` 생성, 고유 `code` 자동 발급
- 교사가 자기 반 목록 확인

### 1-5. 교사 대시보드 (기초)
- 생성한 테스트 목록 + 상태 배지
- 간단 통계 카드(응시 수 등 — 상세는 Phase 3)

**Phase 1 완료 기준:** 교사가 로그인 → 단원 선택 → AI 생성 → 편집 → 테스트 배포까지 끊김 없이 동작.

## Phase 2 (Week 3) — 학생 응시 + 자동 채점

### 2-1. 학생 진입 (로그인 없음)
- 링크(`shareToken`) 또는 클래스 코드 입력 → 이름 입력 → 응시 시작
- `Attempt` 생성(`studentName`, `status: IN_PROGRESS`)

### 2-2. 테스트 플레이어 (태블릿 최적화 + PWA)
- 큰 터치 영역, 진행바, 타이머(시간 제한 시)
- 객관식 라디오, 단답형 입력
- 중간 이탈 대비 답안 임시저장(`Answer`)
- `next-pwa`로 설치 가능 + 기본 오프라인 셸

### 2-3. 자동 채점
- 객관식: `Choice.isCorrect` 매칭, 제출 즉시
- 단답형: `Question.answerKeywords` 키워드 매칭(정규화: 공백/대소문자)
- 결과 집계 → `Attempt.score` / `maxScore`, 학생에게 즉시 결과 표시

**Phase 2 완료 기준:** 학생이 코드/링크로 들어가 응시·제출 → 자동 채점된 점수 확인.

---

## Phase 3 (Week 4) — 분석 & 리포트 (MVP 마무리)

### 3-1. 대시보드 고도화
- 반 평균 점수
- 학생별 점수 테이블(정렬·검색)
- 문제별 정답률 차트(Recharts)
- **약점 단원/학생 하이라이트** (정답률 낮은 단원·하위권 학생 강조)

### 3-2. 리포트 다운로드 (MVP 필수)
- PDF: 반 요약 + 학생별 점수
- Excel: 원본 점수 데이터(필터/가공용)

### 3-3. 문제은행 재사용 UI
- 저장된 문항 검색/필터(단원·난이도·유형)
- 기존 문항을 새 테스트에 다시 담기

**Phase 3 완료 기준 = MVP 완료:** 실제 교사가 "이거 쓰고 싶다"고 말할 수 있는 수준으로 Vercel 배포.

## 작업 진행 규칙
1. 한 번에 한 태스크. 완료 → Preview 링크 → 승인 받고 다음.
2. 각 태스크 시작 전 영향 범위와 변경 파일을 먼저 보고.
3. 스키마 변경 시 마이그레이션 파일 함께 커밋, 설명 명시.
4. 막히면 임의 우회하지 말고 보고 후 결정 요청.
