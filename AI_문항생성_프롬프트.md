# 단평GO — AI 문항 생성 프롬프트 템플릿

> 3학년 수학 단원평가 문항 생성용 · TASK 0(Grok vs Claude 비교) 및 본 기능에 사용
> 핵심 원칙: AI는 초안 생성기일 뿐, **교육과정 적합성과 정답 정확성의 최종 책임은 교사 검수에 있다.**

---

## 1. 사용 방법

1. 아래 **SYSTEM 프롬프트**를 모델의 system 메시지로 고정한다.
2. **USER 프롬프트**의 `{{변수}}`를 교사 입력값으로 치환해 호출한다.
3. 응답은 **JSON only**로 받아 파싱 → DB(`Question`/`Choice`) 저장.
4. 성취기준(`achievementStandard`)이 DB에 있으면 USER 프롬프트의 해당 슬롯에 그대로 주입한다. (정확도 핵심)

---

## 2. SYSTEM 프롬프트

```
당신은 대한민국 초등학교 수학 평가 문항 출제 전문가입니다.
2022 개정 교육과정을 따르며, 초등학교 3학년 학생의 인지 수준에 맞는
단원평가 문항을 출제합니다.

[출제 원칙]
- 3학년 학생이 이해할 수 있는 쉽고 명확한 한국어를 사용한다.
- 한 문항은 하나의 개념만 평가한다.
- 객관식은 4지선다이며, 정답은 정확히 1개다.
- 오답(매력적 오답)은 학생이 흔히 하는 실수를 반영하되, 명백히 틀린 것이어야 한다.
- 보기 간 길이·형식을 비슷하게 맞춰 정답이 티 나지 않게 한다.
- 숫자 계산은 반드시 검산하여 correctValue와 해설이 일치하도록 한다.
- 문화적으로 한국 초등학생에게 자연스러운 소재(이름, 상황)를 쓴다.
- 해설은 풀이 과정을 단계적으로, 3학년이 읽을 수 있게 쓴다.

[계산 정확성 — 필수]
- stem에 포함된 모든 숫자 연산을 직접 계산한 뒤 correctValue를 확정한다.
- 나눗셈 문항: 나누어 떨어지는 수만 사용하거나, 나머지가 있을 경우 반드시 "나머지를 구하시오" 형태로 명시한다.
  예) "28 ÷ 5의 몫을 구하시오"는 나머지가 3이므로 금지. → "28 ÷ 4 = □"처럼 나누어 떨어지는 수 사용.
  예) 나머지를 묻고 싶다면 "28 ÷ 5의 나머지는 얼마입니까?" 형태로 명시.
- 단위 변환: 1L=1000mL, 1kg=1000g, 1m=100cm, 1km=1000m 기준 엄수.
- 결과값이 음수·소수가 되는 연산 금지 (3학년 범위 초과).

[금지]
- 교육과정 범위를 벗어나는 개념(예: 3학년에 미등장 연산) 사용 금지.
- 함정성 표현, 이중 부정, 모호한 문장 금지.
- 특정 교과서·기출문제의 문구를 그대로 베끼지 않는다(자체 창작).
- distractors 에 correctValue 와 동일한 값을 포함하지 않는다.
- distractors 3개가 서로 중복되지 않아야 한다.

[출력 형식]
- 반드시 유효한 JSON만 출력한다. 코드펜스(```)나 설명 문장을 절대 덧붙이지 않는다.
```

## 3. USER 프롬프트 템플릿

> **설계 원칙**: 모델은 `isCorrect` 플래그를 출력하지 않는다.
> 대신 `correctValue`(정답 텍스트)와 `distractors`(오답 3개)를 분리 출력하고,
> 서버 코드가 이를 섞어 `choices[]`를 조립하며 `isCorrect`를 기계적으로 부여한다.
> → stem-정답 불일치를 구조적으로 차단.

```
다음 조건으로 초등학교 3학년 수학 단원평가 문항을 만들어 주세요.

- 학년/학기: 3학년 {{TERM}}학기
- 단원: {{UNIT_NAME}}
- 성취기준(참고): {{ACHIEVEMENT_STANDARD}}
- 문항 수: {{COUNT}}개
- 난이도: {{DIFFICULTY}}  (하=기초 계산/개념 확인, 중=개념 적용, 상=문장제·복합 사고)
- 유형: {{TYPE}}  (MULTIPLE_CHOICE=객관식 4지선다 / SHORT_ANSWER=단답형)

[객관식(MULTIPLE_CHOICE)일 경우] 아래 JSON 스키마로만 응답하세요:

{
  "questions": [
    {
      "type": "MULTIPLE_CHOICE",
      "difficulty": "{{DIFFICULTY}}",
      "stem": "문제 본문",
      "correctValue": "정답 텍스트 그대로 (예: 72, 3팀, 128자루)",
      "distractors": ["오답1", "오답2", "오답3"],
      "explanation": "단계적 풀이 해설"
    }
  ]
}

주의:
- isCorrect 필드는 출력하지 않는다. 보기 조합·순서는 서버가 처리한다.
- distractors 3개는 서로 다른 값이어야 하고, correctValue와도 달라야 한다.
- correctValue는 stem을 직접 계산한 결과여야 한다.
- 나눗셈이 포함된 stem은 반드시 나누어 떨어지는지 확인 후 작성한다.

[단답형(SHORT_ANSWER)일 경우] 아래 JSON 스키마로만 응답하세요:

{
  "questions": [
    {
      "type": "SHORT_ANSWER",
      "difficulty": "{{DIFFICULTY}}",
      "stem": "문제 본문",
      "answerKeywords": ["정답1", "정답1의 다른 표현"],
      "explanation": "단계적 풀이 해설"
    }
  ]
}
```

## 4. 3학년 수학 단원 (시드 참고용)

> 교과서·출판사에 따라 단원명·순서가 다를 수 있으니, 실제 사용 교과서 기준으로 교사가 확정·수정한다. 성취기준 코드는 교사가 어드민에서 입력한다.

**1학기 (대략)**
- 덧셈과 뺄셈 (세 자리 수)
- 평면도형
- 나눗셈
- 곱셈 (두 자리 수 × 한 자리 수)
- 길이와 시간
- 분수와 소수 (기초)

**2학기 (대략)**
- 곱셈 (두/세 자리 수 × 한 자리 수, 두 자리 수 × 두 자리 수)
- 나눗셈
- 원
- 분수
- 들이와 무게
- 자료의 정리(그림그래프 등)

## 5. TASK 0 — 모델 비교 실행 가이드

동일 입력으로 Grok과 Claude를 각각 호출해 비교한다.

**고정 입력 예시**
- 단원: "곱셈 (두 자리 수 × 한 자리 수)"
- 문항 수: 5 / 난이도: 중 / 유형: 객관식

**평가 항목 (각 5점)**

| 항목 | 확인 내용 |
|------|-----------|
| 교육과정 적합성 | 3학년 범위 내 개념만 쓰는가 |
| 정답 정확성 | 계산·정답·해설이 일치하는가 |
| 난이도 적절성 | 3학년 수준에 맞는가 |
| 한국어 자연스러움 | 표현·소재가 자연스러운가 |
| 형식 준수 | JSON 스키마/4지선다 규칙 준수 |

**판정**: 합산 점수 + 오답(틀린 정답·범위 이탈) 발생 빈도를 함께 본다. 정확성 오류가 잦은 모델은 점수와 무관하게 탈락. 결과를 `docs/AI_MODEL_DECISION.md`에 표로 기록하고 기본 모델을 확정한다.

## 6. 품질 가드 (구현)

### stem-정답 불일치 구조적 차단 (2-pass + 재생성)

| 단계 | 내용 |
|------|------|
| **Pass 1** | 모델이 `correctValue` + `distractors` 분리 생성 (`isCorrect` 없음) |
| **Pass 2** | 모델이 stem을 독립 계산해 `correctValue` 검증·교정; 나누어 떨어지지 않는 등 성립 불가 문항은 제거 |
| **보충** | Pass 2에서 제거된 수만큼 재생성 (최대 2라운드) |
| **조립** | 서버 코드가 `correctValue` 기준으로 `isCorrect` 기계적 부여 |

### VERIFY 프롬프트 검수 항목
1. stem을 처음부터 직접 계산해 `correctValue` 불일치 시 교정
2. 나눗셈이 나누어 떨어지지 않는데 몫만 묻는 경우 → 문항 제거
3. 결과가 음수·소수로 3학년 범위 초과 → 문항 제거
4. distractors에 `correctValue`와 동일한 값 → 교체
5. distractors 간 중복 → 교체

### 단원별 주의사항 (`getUnitConstraints`)
단원명을 기반으로 USER 프롬프트에 동적으로 주입되는 추가 제약:
- **나눗셈**: 나누어 떨어지는 수만 사용 또는 나머지 문항 명시 필수
- **곱셈**: 자릿값 올림 검산, 3학년 범위 명시
- **분수**: 분모 10 이하, 약분·통분 금지
- **들이·무게**: 단위 변환 기준, 음수 결과 금지
- **길이·시간**: 단위 변환 기준, 음수 결과 금지
- **덧셈·뺄셈**: 세 자리 수 범위, 음수 결과 금지

### 서버 검증 (`validateQuestion`)
- `choices` 4개 확인
- `isCorrect: true` 정확히 1개 확인
- 보기 중복값 없음 확인 (distractors ≠ correctValue, distractors 간 중복 없음)
- 단답형 `answerKeywords` 1개 이상 확인

### 재시도 정책
- JSON 파싱 실패 시 1회 재시도 → 그래도 실패하면 교사에게 "다시 생성" 노출
- 재생성 2라운드 후에도 부족하면 실제 생성된 수만큼 반환 (호출부가 처리)

### 저장 정책
- 생성 문항은 항상 `isReviewed: false`로 저장 → 교사 검수 전에는 배포 불가 처리(권장)
